# 个人剪辑数据统计网站

这是一个部署到 GitHub Pages 的静态网站，用来记录每天的剪辑数量、复杂片数量和备注，并自动统计今日、本周、本月与今年累计数据。

## 本地运行

```bash
npm install
npm run dev
```

## 构建

```bash
npm run build
```

构建结果会输出到 `dist` 文件夹。

## 数据文件

公开数据保存在：

```text
public/data/editing-records.json
```

网页会读取这个文件作为初始数据。配置 GitHub 令牌后，网页表单会通过 GitHub 接口把新增或修改后的每日数据写回这个文件。

## 登录和编辑权限

网站本身是公开页面，所以任何人都可以打开查看公开数据。真正能保存、导入、删除数据的关键是 GitHub 令牌：

- 没有令牌的浏览器只能查看数据，保存、导入和删除按钮不会写入仓库。
- 用户名、仓库名、分支和令牌会自动保存在当前浏览器本地，下次打开不用重复填写。
- 换手机、换浏览器或清除浏览器数据后，需要重新填写令牌。
- 页面里可以点“清除令牌”，让这台设备重新变成只读状态。

## GitHub 令牌权限

在 GitHub 创建 fine-grained personal access token，也就是细粒度个人访问令牌。

建议权限：

- Repository access：选择这个网站仓库
- Contents：Read and write

令牌只会保存在当前浏览器的本地存储中，不会写进仓库。

## GitHub Pages 部署

仓库推送到 GitHub 后：

1. 打开仓库 Settings，也就是设置。
2. 进入 Pages，也就是静态网页托管。
3. Source 选择 GitHub Actions，也就是 GitHub 自动化流程。
4. 推送到 `main` 分支后会自动构建和部署。

## 腾讯文档导入

腾讯文档里的历史数据可以导出为 `.xlsx`、`.xls` 或 `.csv` 文件，然后在网站的“腾讯文档导入”区域上传。导入前需要先填好 GitHub 令牌，因为导入会直接保存到仓库数据文件。

推荐表头：

```text
日期,剪辑数量,复杂片数量,备注
```

导入时会按日期合并。同一天如果已经有数据，新导入的数据会覆盖旧数据。

导入兼容常见日期格式，例如 `2026-05-23`、`2026/5/23`、`2026年5月23日`、`5/23`，也兼容表格软件里的日期单元格。如果表格前几行是标题说明，网站会在前十行里自动寻找真正的表头。

陪跑团队剪辑记录模板里，视频类别区域中间那一列会被当作复杂片名列。网站会把这一列的片名计入复杂片数量，并在记录列表里用红色标签显示。

## 数据维护

网站里新增了“数据维护”区域：

- 可以按日期范围批量删除数据。
- 可以撤销最近一次导入或删除。
- 可以导出 JSON 备份或 CSV 表格备份。

建议在导入大量腾讯文档数据之前，先导出一次备份。

## 月目标

首页有“月目标”区域，可以填写当前月份的剪辑目标。目标会和剪辑记录一起保存到仓库数据文件里，手机和电脑会读取同一份目标。

月目标统计会自动排除每周一休息日，并显示：

- 当前月目标条数。
- 本月已经完成多少条。
- 还差多少条。
- 当前完成进度百分比。
- 本月平均每天需要剪多少条。
- 从今天到月底剩余工作日里，每天还需要剪多少条。

## 微信推送

个人微信推荐使用 Server 酱。配置后，自动周总结和月总结会推送到微信。

在 GitHub 仓库里进入 Settings，也就是设置，然后进入 Secrets and variables，再进入 Actions，新增密钥：

```text
SERVER_CHAN_SENDKEY
```

值填写你在 Server 酱获得的 SendKey。没有配置这个密钥时，报告仍会生成，只是不会推送到微信。

报告内容包含剪辑总数、复杂片数量、复杂片占比、活跃天数和活跃日均剪辑数。
